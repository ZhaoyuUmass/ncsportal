MobilityFirst

MobilityFirst is a new age internet architecture which is designed to solve the problem of mobility, trustworthiness as current internet architecture cannot address these issues robustly. In the existing internet architecture both identity of endpoint-device and location are represented by a single IP address and that is the reason when a endpoint-device changes from one-network to another network, network stack does not guarantee seamless transmission from one network to another network. Mobile IP solves the problem of mobility up to some extent but that is not scalable for internet.
Mobility First identifies an endpoint by GUID which is hash of the public key of the endpoint. It could have many attributes out of which location (ip address) could be one of those, human readable name could be one of those (as GUID which is generally a hash are difficult to remember) To solve the problem of mobility, location attribute for a GUID should be updated immediately when one moves from one network to another.We cannot use the current method in which DNS records are updated, as it could take couple of hours in updating the ip-address. To update the attributes immediately Mobility First uses Global Name Service (GNS).

Global Name Service (GNS)

Global Name Service is highly scalable distributed name service which stores not only the Ip addresses but also other extensible attributes. It could also be configured as managed dns service and could be used with current internet architecture. GNS internally uses Gigapaxos as key-value database store.In case of GNS keys are GUID and value could be any attributes.

Name Certififying Service:

Suppose a node X (sender) wants to communicate with the node Y (receiver). For communicating with a node, the sender needs to know GUID of Y. Public key is generally a string of fixed length and unique to a node. These are difficult to remember typically. So we associate a name with public key similar to what we do in SSL certificate. This requires a service like Name Certifying Service (NCS) which binds a name and a public key of a given node.If there is no such entity then anyone could come up with a name which would have been used by other node. Also in GNS we could have arbitrary strings representing an idenitity or organization.
